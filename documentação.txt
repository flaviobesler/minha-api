quem cria select, update, insert e delet não é o init_db.py, é a api, o init serve apenas para criar as tabelas.

exemplo mental:
JS → POST /api/products → api.py → INSERT
JS → GET  /api/products → api.py → SELECT
JS → DELETE  /api/products → api.py → DELETE
JS → PUT ou PATCH  /api/products → api.py → UPDATE

INSERT:
INSERT INTO products (nome, valor, categoria) VALUES(?, ?, ?)

SELECT:
'SELECT nome, valor, categoria FROM products'

DELETE:
DELET FROM products WHERE id = ?", (id,)

UPDATE:
UPDATE products SET nome=?, valor=?, categoria=? WHERE id=?
